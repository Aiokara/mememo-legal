mememo プライバシーポリシー

最終更新日: 2026年7月20日

mememo（以下「本アプリ」）は、利用者が日々の出来事を記録し、自分の積み重ねを振り返るためのアプリです。本アプリの運営者 Satoshi Okubo（以下「当方」）は、次の方針で利用者情報を取り扱います。

1. 取り扱う情報

  • 利用者が入力した日記本文、入力日時、任意で選択した気持ち
  • AI分析から得られた出来事、実績、経験値、6つの評価軸、継続情報
  • アプリ内で生成したランダムなインストールID（匿名。氏名・メールとは結び付きません）
  • 購入状態と商品情報（RevenueCatおよびAppleが処理します。カード番号を当方が取得することはありません）
  • 広告の同意状態、広告表示・推定収益などの集計情報
  • 匿名の利用統計：アプリ内の操作イベント、画面遷移、滞在時間、広告の表示・操作、オンボーディングの進行状況など（日記本文・分析結果の内容は含みません。匿名インストールIDに紐づく行動指標のみ）
  • AI処理のモデル名、処理成否、トークン数、推定費用などの運用情報（本文は運用ログに保存しません）
  • 通知（ローカルリマインダー）の設定状態

2. 利用目的

  • 日記の保存、表示、書き出し、削除
  • AIによる出来事・実績・継続・経験値の分析
  • mememo + の購入状態の確認、購入復元、機能提供
  • 広告の表示、同意管理、不正利用の防止、費用管理
  • 利用状況の匿名分析による品質・UXの改善、広告配置の最適化
  • 障害対応、法令上必要な対応

日記本文を、広告の選択・広告プロフィールの作成・利用統計のいずれにも使用しません。

3. 保存場所と外部送信

日記および分析結果は原則として利用者の端末内に保存されます。AI分析を選択した場合に限り、その日に分析対象となる本文と必要最小限の過去の実績・継続情報を、当方のCloudflare Workerを経由してGoogle Gemini APIへ一時的に送信します。当方のサーバーは日記本文を永続保存・ログ保存しません。

次の外部サービスを利用します（各社の取り扱いは各社のプライバシーポリシーにも従います）。

  • Google Gemini API: AI分析（日記本文を送信）
  • Cloudflare Workers / Durable Objects / Analytics Engine: API中継、利用上限、本文を含まない費用・トークンの集計ログ
  • Google AdMob / User Messaging Platform: 広告表示と同意管理（端末・広告関連の識別子を取得する場合があります）
  • RevenueCat: アプリ内購入状態の管理
  • Apple App Store / StoreKit: 決済と購入管理。アプリ内の評価依頼はAppleの仕組み（StoreKitのレビュー）で表示され、当方はどの評価が付いたかを取得しません
  • PostHog: 匿名の利用統計・製品分析（日記本文を送信しません。匿名インストールIDに紐づく行動指標のみ）

外国にある第三者への提供について

上記のうち、次の事業者へのデータ送信は外国にある第三者への提供に当たります。個人情報の保護に関する法律第28条に基づき、以下の情報をお知らせします。

  • Google LLC（AI分析 / 広告）— 所在国: アメリカ合衆国
    米国には日本の個人情報保護法と同等の水準にある包括的な個人情報保護制度は存在せず、分野別の法律と各州法により規律されています。Google は有料版の Gemini API について、送信された内容を自社の生成AIモデルの学習に利用しないことを規約上明示しており、不正利用の検知を目的とするログの保持期間を最大55日間としています。
  • PostHog, Inc.（匿名の利用統計）— 所在国: アメリカ合衆国
    米国の個人情報保護制度は上記のとおりです。当方が PostHog へ送信するのは、日記本文・分析結果を含まない行動指標と、氏名・メールアドレスに結び付かない匿名のインストールIDに限られます。
  • Cloudflare, Inc.（API中継）— 所在国: アメリカ合衆国
    日記本文は中継のみに用い、永続保存・ログ保存を行いません。

AI分析にあたっては、送信前にアプリ内で同意を求めます。同意されない場合、日記は端末内にのみ保存されます。

4. 保存期間

端末内データは、利用者が削除するまで保存されます。設定の「全データを削除」により端末内の日記・分析・実績を削除できます。書き出し機能でJSONまたはテキストとして保存できます。

API中継では日記本文を永続保存しません。利用上限・費用の集計情報、および匿名の利用統計は、運用・不正防止・品質改善に必要な期間に限定して保持します。購入履歴はAppleおよびRevenueCatの規約・法令に従って保持されます。

5. 広告、識別子、トラッキング

本アプリは、同意が必要な地域ではGoogle User Messaging Platformにより選択肢を表示し、広告を要求できる状態になるまで広告SDKを初期化しません。初期版ではIDFAを用いる追跡許可（ATT）を要求しない構成とします。将来、追跡を伴う広告機能を追加する場合は、事前に説明と許可を求め、本ポリシーを更新します。

mememo + が有効な期間はアプリ内広告を表示しません。

6. 利用者の選択

  • 日記を保存するだけで、直ちにAI分析しないことを選べます（送信前に同意を求めます）。
  • 匿名の利用統計は、アプリの「設定 → 利用状況の統計」からいつでも停止できます。
  • 広告のプライバシー選択が必要な場合、アプリ内設定から再表示できます。
  • 通知（ローカルリマインダー）は設定から任意にオン・オフできます。
  • 日記をJSON・テキストで書き出せます。
  • 端末内の全データを削除できます（通知・ロック設定も併せて解除されます）。
  • 購入はApp Storeの設定から解約し、アプリ内から復元できます。

7. 安全性と注意事項

当方は合理的な安全管理措置を講じますが、通信・端末・外部サービスの完全な安全性を保証するものではありません。

  • 日記には、第三者の機微情報、パスワード、秘密鍵、決済番号などを入力しないでください。
  • Face ID等の生体認証によるロックを利用する場合、生体データは端末のOSが管理し、当方が生体データにアクセスすることはありません。

8. 子どもの利用

本アプリは子ども向けとして設計していません。対象年齢および保護者同意については、公開地域の法令とApp Storeの年齢区分に従います。

9. 改定

機能・法令・外部サービスの変更に応じて本ポリシーを改定する場合があります。重要な変更はアプリ内または公開ページで告知します。
